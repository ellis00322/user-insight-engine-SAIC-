# 系统架构说明

本文档描述"可回测用户画像引擎"的四层技术架构及各层职责边界。

## 各层职责说明

### L1 数据层
负责从多个公开渠道采集原始文本数据，统一清洗格式后写入数据湖。关键挑战：跨平台字段对齐、反爬策略、数据时效性管理。

### L2 洞察层
核心模块。通过 LLM 将非结构化评论转化为结构化用户画像标签（人生阶段、购车动机、性别、关注维度等），并聚合为可视化的人群地图。城市线级不经过 LLM，由结构化购车地点字段确定性映射得出。支持按季度切片，追踪用户画像与关注点随产品生命周期的动态漂移（详见 README「画像漂移监测」）。

打标计算完全由飞书多维表格 AI 字段完成——维度提取、情感方向、用户画像（人生阶段/购车动机/性别）都在飞书里由一个推理字段联合判断产出，不引入 Python 或外部 LLM API 参与打标推理，直接满足命题"借助飞书 AI 工具"的要求。Python 只负责读取飞书导出/API 拉取的结构化结果，供 L3 回测计算使用。详见 [`pipeline/llm_tagging/README.md`](../pipeline/llm_tagging/README.md)。产品经理侧只通过飞书多维表格 / Aily 消费最终结果。

### L3 验证层（方案主线）
项目核心创新所在，不是加分项而是方法论校准的基石。以已上市车型（智己 L6）的上市日为时间锚点，只用上市前存在的同级竞品口碑与预热讨论生成预测画像，与上市后真实车主口碑量化比对，输出重合度分数并归因偏差来源。详见 [`backtest/design.md`](../backtest/design.md)。

### L4 应用层
将洞察结果接入飞书 Aily 生态：支持自然语言问答、立项沙盘（新车定位输入 → 预测画像 + 卖点优先级 + 置信度，置信度由 L3 回测准确率背书）、数字人模拟目标用户进行产品沟通测试。
