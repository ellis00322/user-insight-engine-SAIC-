# 回测实验设计

**状态**：提纲完成，具体车型与数据源待确认后填充数值部分  
**定位**：本文档是整个方案的方法论核心，也是验证"可回测用户画像引擎"核心创新的实验蓝图

---

## 1. 实验目标

1. **验证可行性**：用上市前公开数据生成的预测用户画像，能否有效反映上市后真实购买人群？
2. **建立基准**：量化画像准确率，为后续迭代提供可比较的基线数值
3. **归因偏差来源**：识别预测失准的系统性原因（数据偏差 / 产品变化 / 市场扰动 / 模型局限），指导画像方法持续优化
4. **验证方法迁移性**：方法论是否可推广到新车型预测（即不止验证一款车）

---

## 1.1 MVP Demo 简化版回测（比赛演示用）

正式回测需要真实的上市前/后两个时间窗口数据，周期长、样本量大，无法在比赛时限内跑完。Demo 阶段改用以下简化方式，目的是**验证回测链路本身能跑通**，而非产出可信的准确率数值：

1. 手动收集单一竞品车型的 50–100 条公开评论，导入飞书多维表格
2. 用飞书多维表格 AI 字段完成结构化打标（复用 `tagging_prompt.md` 字段定义）
3. 将样本人为切分为两组，分别模拟"预测组"（对应上市前口碑）与"真实组"（对应上市后口碑）
4. 按第 6 节公式计算 POS / DMR / SDA / CBS，验证指标计算逻辑与飞书数据流可以端到端跑通

Demo 输出的分数**不代表真实预测准确率**，方案文档需明确注明这一点。正式规模的回测仍按本文档其余章节的设计执行（≥ 300 条/车型，严格 T0 时间戳切分）。

## 6. 评估指标

### 6.1 画像人群重合度（Profile Overlap Score, POS）

**定义**：上市前预测的用户人群标签分布，与上市后真实车主标签分布的统计相似度。

**计算思路**：
- 对每个离散字段（人生阶段 / 城市线级 / 购车动机），将预测分布 $P$ 与真实分布 $Q$ 视为概率分布
- 使用 Jensen-Shannon 散度衡量差异（取值 0–1，0 表示完全一致）：

$$POS_{dim} = 1 - JS(P \| Q) = 1 - \frac{1}{2}\left[KL(P \| M) + KL(Q \| M)\right], \quad M = \frac{P + Q}{2}$$

- 综合多个维度加权平均：

$$POS_{total} = \sum_{i} w_i \cdot POS_{dim_i}$$

**权重初始设定（待验证调整）**：购车动机 0.4 / 人生阶段 0.3 / 城市线级 0.3

---

### 6.2 关注维度预测准确率（Dimension Match Rate, DMR）

**定义**：上市前识别的 Top-K 用户关注维度，与上市后车主高频提及维度的重合比例。

$$DMR@K = \frac{|\text{predicted\_top}_K \cap \text{actual\_top}_K|}{K}$$

**K 值建议**：K=3（Top3 维度），K=5（Top5 维度）分别计算。

---

### 6.3 情感方向一致率（Sentiment Direction Accuracy, SDA）

**定义**：对同一维度，预测情感倾向（正 / 负 / 中）与真实口碑情感方向是否一致。

$$SDA = \frac{\text{情感方向预测正确的维度数}}{\text{总参与对比的维度数}}$$

---

### 6.4 综合评分（Composite Backtesting Score, CBS）

$$CBS = 0.5 \times POS_{total} + 0.3 \times DMR@5 + 0.2 \times SDA$$

取值 [0, 1]，目标基线：CBS ≥ 0.65（待样本验证后修订）

---

## 7. 偏差归因方法

当某维度 $POS_{dim} < 0.6$ 或 $DMR$ 显著低于预期时，按以下框架逐一排查：

| 偏差类别 | 描述 | 诊断方法 |
|----------|------|----------|
| **A. 数据偏差** | 上市前样本（竞品用户）不代表目标车型真实购买人群 | 对比竞品口碑用户画像 vs 目标车主画像的基线差异 |
| **B. 产品变化** | 实际交付版本与预期/试驾版本存在配置或定价差异 | 查阅上市前后官方配置变更公告 |
| **C. 市场扰动** | 定价调整、竞品上市、政策补贴等外部因素改变了购买结构 | 时间线对齐，检验 T0 前后是否有重大外部事件 |
| **D. 模型局限** | LLM 打标准确率不足，或标签体系粒度不匹配 | 抽取偏差样本人工复核，与 LLM 输出对比 |

归因结论写入每车型的回测报告，格式：`[偏差类别] + [维度] + [具体表现] + [证据]`
