# 回测实验设计

**状态**：目标车型与数据源已确认，指标计算思路已定稿；具体准确率/重合度数值待实验产出后填充（文中一律以 `X%（待实验产出）` 占位，严禁编造）

## 1. 实验目标

1. **验证可行性**：用上市前公开数据生成的预测用户画像，能否有效反映上市后真实购买人群？
2. **建立基准**：量化画像准确率，为后续迭代提供可比较的基线数值
3. **归因偏差来源**：识别预测失准的系统性原因（数据偏差 / 产品变化 / 市场扰动 / 模型局限），指导画像方法持续优化
4. **验证方法迁移性**：方法论是否可推广到新车型预测（即不止验证一款车）

## 1.1 MVP Demo 简化版回测（比赛演示用）

正式回测需要真实的上市前/后两个时间窗口数据，周期长、样本量大，无法在比赛时限内跑完。Demo 阶段改用以下简化方式，目的是**验证回测链路本身能跑通**，而非产出可信的准确率数值：

1. 用 Playwright 爬虫已采集的智己 L6 + 同级竞品（小米 SU7、飞凡 R7 等）小样本口碑，导入飞书多维表格
2. 用飞书多维表格 AI 字段完成结构化打标（复用 `tagging_prompt.md` 字段定义）
3. 将样本按"竞品口碑"（模拟预测组）与"智己 L6 车主口碑"（模拟真实组）划分
4. 按第 6 节公式计算 POS / DMR / SDA / CBS，验证指标计算逻辑与飞书数据流可以端到端跑通

Demo 输出的分数**不代表真实预测准确率**，方案文档需明确注明这一点。正式规模的回测仍按本文档其余章节的设计执行（样本量需达到统计意义规模，严格 T0 时间戳切分；当前样本规模扩充中，具体条数视反爬解决进展确定）。

## 6. 评估指标

### 6.1 画像人群重合度（Profile Overlap Score, POS）

**定义**：预测人群分布（上市前同级竞品口碑推导）与实际人群分布（上市后真实车主口碑推导）的相似度，即各标签维度分布的加权重合度。

**计算思路**：
- 对每个离散字段（人生阶段 / 城市线级 / 购车动机），将预测分布 $P$ 与真实分布 $Q$ 视为概率分布。其中人生阶段、购车动机由 LLM 打标产出（见 `pipeline/llm_tagging/prompts/tagging_prompt.md`）；**城市线级不经过 LLM**，由 ETL 清洗阶段直接用懂车帝结构化字段"购车地点"确定性映射得出（覆盖率 85%，比 LLM 从文本推断的约 15% 可靠得多，见 `pipeline/etl/clean_reviews.py`）——三个字段口径不同但都是"结构化的人群分布"，可以一并纳入 POS 计算
- 使用 Jensen-Shannon 散度衡量差异（取值 0–1，0 表示完全一致）：

$$POS_{dim} = 1 - JS(P \| Q) = 1 - \frac{1}{2}\left[KL(P \| M) + KL(Q \| M)\right], \quad M = \frac{P + Q}{2}$$

- 综合多个维度加权平均：

$$POS_{total} = \sum_{i} w_i \cdot POS_{dim_i}$$

**权重初始设定（待验证调整）**：购车动机 0.4 / 人生阶段 0.3 / 城市线级 0.3

**性别（gender）暂不纳入 POS 计算**：车评文本里能间接推断性别的线索覆盖率约 11%，虽然和人生阶段（约 12%）相当、字段本身保留在打标结果里，但样本量对回测这种量化对比来说偏薄，先只作为人群地图的辅助下钻维度，不计入核心回测指标；后续样本量上来后可以评估要不要补进 POS。

---

### 6.2 卖点匹配率（Dimension Match Rate, DMR）

**定义**：预测阶段识别的 Top-3 卖点，与目标车型上市后车主口碑中实际高频提及的正面维度 Top-3 的命中数量占比。

**计算思路**：
- **预测侧**：从同级竞品口碑的 LLM 打标结果中，选出提及频次最高、且情感倾向为正面的 Top-3 维度，作为"预测卖点"
- **真实侧**：从目标车型上市后车主口碑的 LLM 打标结果中，同样筛选情感倾向为正面且提及频次最高的 Top-3 维度，作为"实际高频正面维度"
- **命中数**：两个 Top-3 集合的交集大小，除以 3 得到匹配率

$$DMR@3 = \frac{|\text{predicted\_top3\_selling\_points} \cap \text{actual\_top3\_positive\_dims}|}{3}$$

同时计算更宽松的 Top-5 版本（DMR@5）作为参考指标，用于 6.4 节综合评分。

---

### 6.3 情感方向一致率（Sentiment Direction Accuracy, SDA）

**定义**：对同一维度，预测情感倾向（正 / 负 / 中）与真实口碑情感方向是否一致。

$$SDA = \frac{\text{情感方向预测正确的维度数}}{\text{总参与对比的维度数}}$$

---

### 6.4 综合评分（Composite Backtesting Score, CBS）

$$CBS = 0.5 \times POS_{total} + 0.3 \times DMR@5 + 0.2 \times SDA$$

## 7. 偏差归因方法

当某维度 $POS_{dim} < 0.6$ 或 $DMR$ 显著低于预期时，按以下框架逐一排查：

| 偏差类别 | 描述 | 诊断方法 |
|----------|------|----------|
| **A. 数据偏差** | 上市前样本（竞品用户）不代表目标车型真实购买人群 | 对比竞品口碑用户画像 vs 目标车主画像的基线差异 |
| **B. 产品变化** | 实际交付版本与预期/试驾版本存在配置或定价差异 | 查阅上市前后官方配置变更公告 |
| **C. 市场扰动** | 定价调整、竞品上市、政策补贴等外部因素改变了购买结构 | 时间线对齐，检验 T0 前后是否有重大外部事件 |
| **D. 模型局限** | LLM 打标准确率不足，或标签体系粒度不匹配 | 抽取偏差样本人工复核，与 LLM 输出对比 |

归因结论写入每车型的回测报告，格式：`[偏差类别] + [维度] + [具体表现] + [证据]`
