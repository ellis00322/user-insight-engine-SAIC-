# 智己L6 Demo 回测报告

> **性质说明**：本报告是 MVP Demo 阶段的链路验证产出，目的是证明"打标 → 回测计算 → 偏差归因"这条链路能端到端跑通，**不代表最终可信的画像准确率**。样本规模、打标质量均未达到正式规模标准，具体原因见下文。参见 [`backtest/design.md`](../design.md) 第 1.1 节对 demo 阶段定位的说明。

---

## 1. 数据来源

- **目标车型**：智己L6（series_id 9660），验证基准 253 条记录
- **竞品参照系**：小米SU7、飞凡R7、ZEEKR 007、小鹏P7、MG领航PHEV、深蓝SL03（series_id 6187/4980/9128/3352/4736/5306），预测输入取其中 `phase=pre` 记录，602 条
- **打标方式**：飞书多维表格 AI 字段（`profile_json` 推理字段 + 抽取字段），详见 [`pipeline/llm_tagging/README.md`](../../pipeline/llm_tagging/README.md)
- **数据文件**：`样本分析demo tagged.xlsx`（1296 条原始记录）

## 2. 打标质量问题与处理过程

正式跑回测前，人工抽检 `profile_json` 发现严重的格式一致性问题，处理过程如实记录：

| 问题 | 规模 | 根因（推测，无法直接查看飞书 AI 字段配置） |
|---|---|---|
| `profile_json` JSON 结构不统一 | 183 种不同 key 组合，最常见的一种仅占 28% | 提示词大概率没有锁死输出 Schema，且中途可能被反复修改过，不同批次跑出不同格式 |
| JSON 解析失败 | 69 条（5.3%） | 同上，格式混乱到部分输出连合法 JSON 都不是 |
| 维度名称不闭集 | 2292 种不同说法（应为 17 类闭集） | 提示词未强制维度枚举 |
| 情感表述不闭集 | 30+ 种说法（应为"正面/负面/中性"三类） | 提示词未强制情感枚举，部分记录把推理过程文字当成情感值输出 |
| `user_profile`（画像）字段 | 首次统计时 0% 可恢复 | 大部分批次的提示词实际未要求输出画像字段 |

**处理方式**（规则化清洗，非新一轮 AI 推理）：

1. 维度名称按关键词规则归一到 17 类闭集，映射不上的归入"其他"（30% 记录，其中部分本就是"购车决策""综合评价"这类非产品属性的评价性文字，归"其他"合理）
2. 情感表述按关键词规则归一到三类，无法判断的**直接丢弃**（32.5%），不猜测
3. `evidence` 校验：非空时必须是原文真实子串，不通过的字段置空（校验通过率 86.75%，好于 v1 智谱方案的 26.8%）
4. `life_stage`/`purchase_motivation`/`purchase_stage`/`gender` 在原有基础上，用关键词规则在**原文全文**（不依赖 `profile_json`）上补全，补全后覆盖率：`purchase_motivation` 10.6%→36.8%，`purchase_stage` 39%→60.0%，`gender` 1.3%→5.3%，`life_stage` 2.5%→3.8%
5. 清洗结果写回 `profile_json_cleaned` 列，`positive_dimensions`/`negative_dimensions`/`neutral_dimensions` 三列据此重新生成

## 3. 回测指标结果

| 指标 | 数值 | 参与计算的样本量 |
|---|---|---|
| POS_total | 0.965 | — |
| ├ purchase_motivation | 0.953（权重 0.4） | 预测组 263/602 有值，实际组 84/253 有值 |
| ├ life_stage | 0.952（权重 0.3） | 预测组 34/602 有值，**实际组仅 4/253 有值** |
| ├ city_tier | 0.995（权重 0.3） | ETL 确定性映射，样本量充分 |
| DMR@3 | 0.333 | 预测 Top3：{内饰质感、动力性能、外观设计}；实际 Top3：{底盘操控、续航表现、外观设计} |
| DMR@5 | 0.600 | 预测 Top5：{内饰质感、续航表现、动力性能、空间表现、外观设计}；实际 Top5：{续航表现、底盘操控、空间表现、辅助驾驶、外观设计} |
| SDA | 1.000 | 16 个共同维度 |
| CBS | **不出具** | 见下方说明 |

**关于 CBS**：`backtest/design.md` 第 6.4 节的规则是"POS/DMR/SDA 任一项样本不足，CBS 不能硬凑"。这次 POS 里的 `life_stage` 分项只有 4 个有效样本、SDA 存在满分假象，两项都不满足"可信"这个前提，所以本报告**不给出 CBS 汇总分**，避免一个单一数字脱离上下文被引用，盖过第 4 节列出的这些保留意见。

## 4. 每个数字的可信度评估（偏差归因）

按 `backtest/design.md` 第 7 节框架逐一归因：

### POS = 0.965 —— 不可直接采信为"画像预测准确率"

- **`life_stage` 分项（0.952）不可信**：实际组仅 4 条非空样本，JS 散度基于 4 个数据点计算，统计意义不足，归因为 **D. 模型局限**（打标覆盖率过低，非方法论问题）
- `purchase_motivation`（0.953）相对可参考：实际组 84 条样本，量级尚可，但仍建议扩大样本后复核
- `city_tier`（0.995）可信：ETL 确定性映射，非 AI 推断，样本量充分

### DMR@3 = 0.333 / DMR@5 = 0.600 —— 已排除已知污染源，相对可信

初次计算时"其他"维度（打标时的兜底桶）混入 Top3/Top5 排名，造成预测组与实际组"假命中"。已在 `backtest/compute_metrics.py` 的 `positive_dimension_counts`/`majority_sentiment_per_dimension` 中排除"其他"参与排名，修复前后对比：

| | 修复前 | 修复后 |
|---|---|---|
| DMR@3 | 0.667（虚高，"其他"假命中） | 0.333（真实重合仅"外观设计"一项） |
| DMR@5 | 0.400（真实维度被"其他"挤出榜单） | 0.600（续航表现/空间表现/外观设计三项真实重合） |

归因：**D. 模型局限**（打标维度枚举未严格约束导致"其他"桶过大），已通过代码修复。

### SDA = 1.000 —— 满分不正常，暂不采信

16 个共同维度全部情感方向一致，大概率是清洗阶段丢弃了 32.5% 情感表述模糊的记录后，剩余样本本身正负面倾向清晰、且车评整体偏正面导致的假象，不代表方法论真的 100% 准确。归因：**A. 数据偏差**（清洗时的样本选择偏差）+ **D. 模型局限**（原始情感标注枚举未约束）。这一项尚未修复，需要重新用严格 Schema 的提示词跑一遍干净数据才能拿到可信的 SDA。

## 5. 结论

1. **链路验证通过**：飞书打标 → 数据清洗 → 回测计算 → 偏差归因，完整跑通一次，证明方法论本身可执行
2. **当前数值不代表最终准确率**：POS 的 `life_stage` 分项、SDA 的满分结果均存在明确的样本量/数据偏差问题，需要在 `profile_json` 提示词修复为严格 Schema 版本、重新跑一遍全量数据后，才能拿到可信的正式回测数值
3. **过程本身即偏差归因能力的证明**：本次发现并修复的"其他"维度污染问题，直接验证了 `backtest/design.md` 第 7 节偏差归因框架的可操作性——不是纸面设计，是真的能定位、能修、能复核的机制
4. **下一步**：按 `pipeline/llm_tagging/prompts/feishu_bitable_fields.md` 里的严格 Schema 提示词重新跑一遍飞书 `profile_json` 全量数据，替换本次 demo 数据，产出正式回测数值
